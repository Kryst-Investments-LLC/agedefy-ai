# Security Policy

## Supported versions

| Version  | Supported |
|----------|-----------|
| `1.x`    | ✅         |
| `0.x`    | ❌ (pre-release) |

## Reporting a vulnerability

Please **do not** open public issues for security reports.

- Email: **security@agedefy.ai**
- PGP key fingerprint: `0000 0000 0000 0000 0000  0000 0000 0000 0000 0000`
  (full key published on https://agedefy.ai/.well-known/security.txt)

We aim to acknowledge within **2 business days** and provide a fix or
mitigation timeline within **10 business days**. Coordinated disclosure
is encouraged; we credit reporters in the [CHANGELOG](CHANGELOG.md)
unless anonymity is requested.

## In-scope

- Agent specs, workflows, legal rules, tooling under this repo.
- CI configuration that may leak credentials or downgrade verification.

## Out of scope

- The Next.js product layer (see its own SECURITY.md in that repo).
- Third-party integrations (Stripe, OpenAI, Anthropic, Grok, Nodemailer).

## Sensitive data

- This repo must never contain real user PHI, API secrets, or
  customer-identifying data. Pre-commit `gitleaks` is required.
- Synthetic / fictional data only in tests, fixtures, and docs.

## Release integrity

- All tags are signed (`git tag -s`).
- Release artifacts ship with an SBOM (`syft`) and `cosign` signature
  generated by `.github/workflows/release.yml`.

## Compliance crosswalk

The platform's legal-rules layer is the operational implementation of
HIPAA, GDPR, UK GDPR, MHRA, TGA, ANVISA, LGPD, PIPEDA, DPDP, and
state-level rules. See [agents/legal-rules/](agents/legal-rules/) and
[docs/legal/](docs/legal/).
